# Solution strategy

This section covers the quality goals, a mapping to quality scenarios, and a solution approach.
The listed quality goals come from [the introduction section](./01-introduction-and-goals.md).
Quality scenarios mentioned here are expanded in the [quality](./quality/README.md) section.

| **Quality Goal**                                                            | **Scenario**                                                                             | **Solution Approach**                                                                                                                      |
| --------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **Robust workflow for content summarization and podcast generation**        | The summarization pipeline fails due to an API timeout or malformed input.               | Use Dapr Workflow with retry policy and monitoring to make the workflow more robust.                                                       |
|                                                                             | The podcast generation workflow fails due to an API timeout or malformed input.          | Use Dapr Workflow with retry policy and monitoring to make the workflow more robust.                                                       |
|                                                                             | Generated podcast content audio makes no sense.                                          | Create integration tests to verify audio matches summary segments; use checksums or text-to-speech markers to validate alignment.          |
|                                                                             | Generated summary content is nonsensical or the worklow breaks due to LLM changes.       | Provide prompt tests to verify the quality of the LLM generated content.                                                                   |
| **Accessible solution for collecting interesting content from the web**     | Users want to add content via bookmarklets, browser extensions, or a mobile app.         | Provide a simple API endpoint and SDK or browser extension to allow easy submission of URLs. Use authentication to track user submissions. |
|                                                                             |                                                                                          | Provide integration with GMail API to monitor an external mailbox for content submissions and save them in the content API as normal.      |
|                                                                             | A content source (e.g. a website or RSS feed) changes its layout or becomes unavailable. | Monitor text extraction behavior and alert to text parsing failures with content source included in the alert.                             |
| **Easily modifiable because we expect improvements in the AI capabilities** | A newer LLM becomes available and needs to be integrated.                                | Use Dapr Agent Framework to abstract access to LLMs.                                                                                       |
|                                                                             | Developers want to change prompt templates or tweak generation parameters.               | Store prompts and model settings in a prompt file, enabling non-code changes. Version control all prompt edits.                            |
